//
The aim of this document is to compare the c++ math library
according to solve typical FEA problem (especially rotor system,
i.e. a skew-symmetric (or antisymmetric or antimetric[1]) matrix))
//
Main criteria are:
1. learning curve
2. only-header?
3. open-source?
4. det for sym and and non-sym matrix?
5. inverse matrix?
5. lsolve for sym and band matrix?
5.1. LU decomposition
6. eigenvecs for sym and band matrix?
7. lsolve for non-symmatrix?
8. eigenvecs for nn-sym matrix?
8.1 complex value?
9. runtime size for matrix
10. perfomance
